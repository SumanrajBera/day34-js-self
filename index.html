Features and Techniques

Event Delegation

Usage/Info: This used when you have multiple items wiith some kind of event grouped together. So rather than selecting one at a time we delegate and select the parent to have the event listener. This based on Event bubbling.

→ Click on list item to strikethrough:

  • Shop for school
  • Buy some veggies
  • Wash your car
  • Fill the gas tank

Event Bubbling

Usage/Info: This happens almost everytime except for events which do not bubble. The inner workings are that events are triggered in a sequence from the element that triggered till it's last parent which has event.(Siblings not included)

→ Click on the button in center and see all the events are triggered

A has a event

B has a event

C doesn't have a event

Event Capturing

Usage/Info: This happens before event bubbling and is turned off by default. This is exact opposite of bubbling it means this will trigger the top level event first then go towards the event triggering element. To turn this on pass a boolean true as an argument in even listener.

Note: There are two phases in an event flow first phase is event capturing then second phase event bubbling happens.

→ Click on the button in center and see all the events are triggered

B has a event (Triggers second)

C has an event (Triggers fourth)
